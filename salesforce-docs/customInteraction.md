# Read Custom Interaction Data from Package Usage Logs

## Overview

After instrumenting your managed package with Custom Interactions, App Analytics records those events in Package Usage Logs.

Custom Interaction logs allow ISV partners to answer questions such as:

* Which features are being used most frequently?
* How do users progress through a workflow?
* Which business processes are completed successfully?
* Which UI components receive the most engagement?
* Which features are no longer being used?

Custom Interactions are surfaced as a specialized record type within App Analytics Package Usage Logs.

---

# Prerequisites

Before custom interaction data appears in Package Usage Logs:

* App Analytics must be enabled.
* The managed package must have passed Security Review.
* The package must contain instrumented Apex code using `IsvPartners.AppAnalytics.logCustomInteraction()`.
* Subscribers must be running a package version that contains the instrumentation.

Example:

```apex
public enum MyCustomInteractions {
    CREATE_INVOICE,
    COMPLETE_CHECKOUT
}

IsvPartners.AppAnalytics.logCustomInteraction(
    MyCustomInteractions.CREATE_INVOICE
);
```

---

# Where Custom Interactions Appear

Custom Interactions are stored within Package Usage Logs.

Workflow:

```text
User Action
    ↓
Managed Package Apex
    ↓
logCustomInteraction()
    ↓
App Analytics
    ↓
Package Usage Log
    ↓
Analysis / Reporting
```

Every successful call to `logCustomInteraction()` generates a corresponding Package Usage Log record.

---

# Identifying Custom Interaction Records

To locate custom interaction events, filter Package Usage Logs using:

```text
log_record_type = CustomInteraction
```

and

```text
custom_entity_type = CustomInteractionLabel
```

These two fields uniquely identify Custom Interaction events within the larger App Analytics dataset.

---

# Core Fields

## log_record_type

Value:

```text
CustomInteraction
```

Indicates the log entry was generated by the Custom Interaction framework.

---

## custom_entity_type

Value:

```text
CustomInteractionLabel
```

Indicates that the event represents a user-defined interaction label.

---

## custom_entity

Contains the Apex enum value that was logged.

Example:

```text
MyCustomInteractions.CREATE_INVOICE
```

or

```text
MyCustomInteractions.COMPLETE_CHECKOUT
```

This field is the primary field used for feature adoption reporting.

---

## interaction_id_token

Contains a hashed and tokenized version of:

* Record ID
* UUID
* Correlation ID

when one was provided during logging.

Example:

```apex
Id invoiceId = invoice.Id;

IsvPartners.AppAnalytics.logCustomInteraction(
    MyCustomInteractions.CREATE_INVOICE,
    invoiceId
);
```

App Analytics stores a tokenized value rather than the original identifier to protect subscriber data.

---

# Example Log Record

Example JSON representation:

```json
{
  "log_record_type": "CustomInteraction",
  "custom_entity_type": "CustomInteractionLabel",
  "custom_entity": "MyCustomInteractions.CREATE_INVOICE",
  "interaction_id_token": "7abf2d19..."
}
```

Key fields:

| Field                | Value                               |
| -------------------- | ----------------------------------- |
| log_record_type      | CustomInteraction                   |
| custom_entity_type   | CustomInteractionLabel              |
| custom_entity        | MyCustomInteractions.CREATE_INVOICE |
| interaction_id_token | Tokenized identifier                |

# Source Location Metadata

Custom Interaction logs also include Apex execution metadata.

Common fields include:

```text
class_name
method_name
line_number
```

Example:

```text
class_name  = InvoiceService
method_name = createInvoice
line_number = 42
```

These fields help determine exactly where in Apex code the interaction was generated.

# Querying Feature Usage

To determine how often a feature is used:

Filter:

```text
log_record_type = CustomInteraction
```

```text
custom_entity =
MyCustomInteractions.CREATE_INVOICE
```

Result:

```text
Count all matching records
```

This provides total usage volume for that feature.

---

# Tracking User Journeys

Custom Interactions become especially powerful when a shared identifier is supplied.

Example:

```apex
String journeyId = Crypto.getRandomUUID();

IsvPartners.AppAnalytics.logCustomInteraction(
    MyCustomInteractions.START_CHECKOUT,
    journeyId
);

IsvPartners.AppAnalytics.logCustomInteraction(
    MyCustomInteractions.ADD_PAYMENT_METHOD,
    journeyId
);

IsvPartners.AppAnalytics.logCustomInteraction(
    MyCustomInteractions.COMPLETE_CHECKOUT,
    journeyId
);
```

All events share the same tokenized interaction identifier.

This allows:

* Funnel analysis
* Journey completion reporting
* Conversion tracking
* Process monitoring

---

# Failure Records

Sometimes App Analytics cannot log a Custom Interaction.

These events appear as:

```text
custom_entity_type = CustomInteractionFailure
```

Instead of:

```text
CustomInteractionLabel
```

Example:

```text
OVER_CALL_LIMIT
```

This occurs when more than 50 Custom Interactions are logged during a single transaction.

---

# Common Failure Codes

## OVER_CALL_LIMIT

Cause:

```text
More than 50 interactions logged
during a single user request
```

Example:

```apex
for(Account a : accounts){

    IsvPartners.AppAnalytics.logCustomInteraction(
        MyCustomInteractions.PROCESS_RECORD
    );
}
```

Avoid placing Custom Interaction logging inside loops.

# Relationship to Monthly Summaries

Custom Interaction data is also aggregated into Package Usage Summary datasets.

Summary records use:

```text
custom_entity_type =
CustomInteractionLabel
```

and

```text
custom_entity_type =
CustomInteractionFailure
```

This provides monthly rollups without requiring detailed log analysis.

---

# Debugging During Development

Package Usage Logs are not available in Developer Edition orgs.

During development:

1. Enable Apex debug logs.
2. Set log level to:

```text
FINE
```

3. Search for:

```text
APP_ANALYTICS_FINE
APP_ANALYTICS_WARN
APP_ANALYTICS_ERROR
```

These messages confirm whether the interaction was successfully processed.

# Best Practices

## Create Meaningful Labels

Good:

```apex
CREATE_INVOICE
EXPORT_REPORT
COMPLETE_CHECKOUT
```

Avoid:

```apex
BUTTON_1
BUTTON_2
BUTTON_3
```

---

## Analyze Business Outcomes

Focus on:

* Workflow completion
* Adoption milestones
* Feature usage
* Customer engagement

rather than low-level UI clicks.

---

## Correlate Related Events

Use:

```apex
Id
```

or

```apex
UUID
```

to group related interactions.

---

## Monitor Failure Events

Track:

```text
CustomInteractionFailure
```

to ensure instrumentation remains healthy.

# Summary

Custom Interaction events are stored in Package Usage Logs as:

```text
log_record_type = CustomInteraction
```

with:

```text
custom_entity_type = CustomInteractionLabel
```

The most important fields are:

| Field                | Purpose              |
| -------------------- | -------------------- |
| custom_entity        | Interaction label    |
| interaction_id_token | Correlation token    |
| class_name           | Source Apex class    |
| method_name          | Source Apex method   |
| line_number          | Source code location |

Together, these fields allow ISV partners to measure feature adoption, analyze user journeys, monitor business processes, and make data-driven product decisions using App Analytics.
